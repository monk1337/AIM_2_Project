# OOD Evaluation of Finetuned WiLoR

Evaluates the POV-Surgery-finetuned WiLoR hand-pose estimator on two out-of-distribution
datasets (Aria smart-glasses recordings, AIxSuture surgical videos) plus visualization
tooling for comparing raw vs. finetuned predictions on the same frames.

For results and analysis see `RESULTS.md`.

## Prerequisites

- Finished the POV-Surgery finetuning pipeline under `AIM_2_Project/finetuning/`
  (or downloaded the pretrained finetuned checkpoint — see below).
- `aim2` conda env with all deps from `eval/requirements.txt` + `datasets` from HF.

## Required downloads

Nothing in this directory is committed with actual data — everything below is
gitignored and must be fetched before running any script.

### 1. Finetuned WiLoR checkpoint (required for `--ckpt-path`)

```
finetuning/output/checkpoints/last.ckpt  (7.2 GB, step 50k)
```

Host: private HuggingFace model repo `suyc21/wilor-pov-surgery-ft`
(ask Yuchang for read access if you don't have it already).

```bash
mkdir -p finetuning/output/checkpoints

huggingface-cli download suyc21/wilor-pov-surgery-ft last.ckpt \
    --local-dir finetuning/output/checkpoints --local-dir-use-symlinks False
```

The eval scripts look for `last.ckpt` at that exact path by default.

If you want to reproduce the training instead, see `finetuning/README.md`.

## Reproduction

```bash
source /mnt/ssd/yuchang/miniforge3/bin/activate aim2
cd AIM_2_Project/ood_eval

# 1. Build common-format datasets (one-time)
python adapters/aria_to_common.py           # 784 samples, 481 images, ~800 MB
python adapters/aixsuture_to_common.py      # downloads 4.4 GB, cleans up

# 2. Run evaluation — raw WiLoR
python run_eval_ood.py --dataset-dir common_format/aria \
    --output-dir results/aria_raw --device cuda
python run_eval_ood.py --dataset-dir common_format/aixsuture \
    --output-dir results/aixsuture_raw --device cuda

# 3. Run evaluation — finetuned WiLoR
python run_eval_ood.py --dataset-dir common_format/aria \
    --output-dir results/aria_finetuned \
    --ckpt-path ../finetuning/output/checkpoints/last.ckpt --device cuda
python run_eval_ood.py --dataset-dir common_format/aixsuture \
    --output-dir results/aixsuture_finetuned \
    --ckpt-path ../finetuning/output/checkpoints/last.ckpt --device cuda

# 4. Per-model overlays
for DS in aria_raw aria_finetuned aixsuture_raw aixsuture_finetuned; do
    python visualize_ood.py \
        --results results/${DS}/wilor_ood_results.json \
        --dataset-dir common_format/${DS%_*} \
        --output-dir results/${DS}/figures/sample_overlays --n-each 2
done

# 5. Raw-vs-finetuned side-by-side on same frames
python compare_raw_vs_ft.py \
    --raw results/aria_raw/wilor_ood_results.json \
    --ft  results/aria_finetuned/wilor_ood_results.json \
    --dataset-dir common_format/aria \
    --output-dir results/compare_aria \
    --n-wins 4 --n-losses 4 --rank-by p2d

python compare_raw_vs_ft.py \
    --raw results/aixsuture_raw/wilor_ood_results.json \
    --ft  results/aixsuture_finetuned/wilor_ood_results.json \
    --dataset-dir common_format/aixsuture \
    --output-dir results/compare_aixsuture \
    --n-spread 8
```

Expected wall-clock: ~5 min (Aria eval, 784 samples) + ~30 sec (AIxSuture)
+ ~50 min (AIxSuture download, one-time) on a single modern GPU.

## Common-format sample schema

```python
{
  "frame_id":    "PR83/000123/right",       # unique key
  "image_path":  "images/PR83_000123.png",  # relative to samples.pkl
  "K":           np.float32 (3, 3),         # per-sample intrinsics
  "joints_3d":   np.float32 (21, 3) | None, # OpenPose order, meters
  "joints_3d_frame_trustworthy": bool,      # gates absolute MPJPE/PVE
  "vertices_3d": np.float32 (778, 3) | None,
  "joints_2d":   np.float32 (21, 2) | None, # pixels, OpenPose order
  "bbox":        np.float32 (4,),           # xyxy hand region
  "is_right":    bool,                      # runner flips images when False
  "hand_side":   "left" | "right",
}
```

`run_eval_ood.py` runs WiLoR crop-regress on each sample, handles left-hand
flip+un-flip, gates metrics on GT availability, and writes both metrics AND
predictions (in original image coords) into the output JSON so visualization
needs no re-inference.

## Gotchas / non-obvious bits

- **Aria image rotation**: the stored 1408×1408 JPEGs are in sensor-native
  orientation; the adapter rotates them 90° CW so they align with
  `intrinsics` and `joints_2d`. All downstream code consumes rotated images.
- **Aria `eval_joints_3d`** is NOT in the projection camera frame. 3D
  PA-MPJPE is meaningful but absolute MPJPE/PVE are disabled via
  `joints_3d_frame_trustworthy=False`.
- **Left hands on Aria** (346/784): model is right-only, so the runner
  horizontally flips image+bbox, runs WiLoR, then un-flips predictions back
  to original image coords.
- **AIxSuture bbox**: placeholder 60% center crop since there are no hand
  annotations. Expect the model to hallucinate skeletons on frames where
  the hand is off-centre.
- **`compare_raw_vs_ft.py --rank-by p2d`** (default) matches what the 2D
  visualization shows. Using `--rank-by pa_mpjpe` can disagree visually
  because 3D PA-MPJPE on Aria ignores projection-frame alignment.
